# v0.0.5 Android release — build & ship handoff

Everything on the `mobile-ui` branch is **code-finalized, versioned (0.0.5),
and tested on the non-CUDA matrix** (see `RELEASE_NOTES_v0.0.5.md`). The only
remaining step is producing and shipping the **Android APK/AAB**, which must
run in a **Linux/WSL2** (or CI) environment — a clean Windows
`cargo tauri android build` is blocked by Smart App Control on unsigned crate
build-scripts, and per-machine SAC disable is one-way.

## One-time toolchain (Ubuntu/WSL2)

```bash
# Rust Android targets
rustup target add aarch64-linux-android armv7-linux-androideabi \
                  i686-linux-android x86_64-linux-android

# JDK + Tauri CLI
sudo apt install -y openjdk-17-jdk
cargo install tauri-cli --version '^2'   # or: npm i -g @tauri-apps/cli

# Android SDK + NDK (sdkmanager, or Android Studio)
#   export ANDROID_HOME=$HOME/Android/Sdk
#   export NDK_HOME=$ANDROID_HOME/ndk/<version>
sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0" "ndk;26.1.10909125"
```

Build prerequisites for the native crates (llama-cpp / ggml, surrealdb):
`clang`, `cmake`, `libclang-dev`, `protobuf-compiler` (if prompted). The
`mobile` feature set excludes the CUDA, cpal-voice, native-tls-email, and
embedded-browser paths, so no CUDA toolkit is needed for the Android build.

## Build steps

```bash
cd sovereign

# 1. Frontend (resolver script is cwd-agnostic; tauri.android.conf.json
#    points beforeBuildCommand at it). Sanity-check it standalone first:
node scripts/tauri-build-frontend.cjs

# 2. If gen/android is stale relative to tauri.conf.json, re-init once:
#    cargo tauri android init      # regenerates gradle/manifest; re-apply
#                                   # the manifest perms + plugin copies if so
# 3. Debug APK
cargo tauri android build --debug --target aarch64
#    → crates/sovereign-app/gen/android/app/build/outputs/apk/.../*.apk

# 4. Release (signed) — set up a keystore and signingConfig in
#    app/build.gradle.kts, then:
cargo tauri android build --target aarch64           # AAB/APK release
```

Confirm `versionName` in the built artifact is `0.0.5` (driven by
`tauri.conf.json` → `tauri.android.versionName`); bump `versionCode` per store
upload.

## Smoke test on device/emulator

```bash
adb install -r path/to/sovereign-0.0.5.apk
```

Manual checks worth doing (not covered by unit tests):
- Onboarding → unlocked session; AccountKey migration runs once on a v0.0.4 store.
- Pairing: host shows QR, second device scans + PIN → imports account key → both decrypt the same graph.
- Sync: edit on device A, foreground device B → row appears (LWW); taskbar sync indicator updates.
- Share-sheet: share a URL/text from another app → SharePickerSheet → save to a thread.
- Canvas: vertical = time, pinch zoom, density strip at deep zoom; mobile chat sheet + voice.
